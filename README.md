# Expensea

Expensea is an open-source expense management app for teams. It combines shared expenses, approvals, reimbursements, budgets, settlements, reporting, recurring expenses, notifications, and public read-only sharing in one Supabase-backed workspace.

## Features

- Multi-team workspaces with active team switching.
- Team roles: owner, admin, and viewer.
- Individual and team expenses with category, payer, assignee, payment status, and approval workflow.
- Shared expenses with equal or custom participant splits.
- Reimbursements for approved expenses.
- Settlement records and member ledgers for shared balances.
- Team and category budgets with warning and over-budget states.
- Reports, analytics, dashboard widgets, saved views, and CSV/Excel/PDF-style exports.
- Recurring expenses generated by a cron route.
- Realtime notifications and activity logs.
- Public team and member summary pages when enabled by team settings.

## Tech Stack

- Next.js App Router, React, TypeScript
- Supabase Auth, Postgres, Row Level Security, Realtime
- Tailwind CSS, Radix UI, shadcn/ui-style components
- React Hook Form, Zod, TanStack Table, Recharts, Framer Motion

## Requirements

- Node.js 22.x or newer
- npm
- Supabase project with Auth and Postgres enabled

## Installation

```bash
git clone <repository-url>
cd expensea
npm install
```

Create `.env.local` and set:

```env
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=
NEXT_PUBLIC_APP_URL=http://localhost:3000
CRON_SECRET=
```

Apply migrations in `supabase/migrations/` in filename order, then configure Supabase Auth:

- Enable email/password auth.
- Set the Site URL to `NEXT_PUBLIC_APP_URL`.
- Add `{NEXT_PUBLIC_APP_URL}/auth/callback` as a redirect URL.

Seed demo data:

```bash
npm run seed:demo
```

Run the app:

```bash
npm run dev
```

Open `http://localhost:3000`.

Demo login:

```text
owner@expensea.app
password123
```

## Scripts

| Command | Description |

| `npm run dev` | Start the development server. |
| `npm run build` | Build the production app. |
| `npm run start` | Start the production build. |
| `npm run lint` | Run ESLint. |
| `npm run seed` | Seed demo data without clearing existing demo teams unless needed. |
| `npm run seed:demo` | Reset and populate a complete demo workspace. |
| `npm run db:reset` | Remove seeded demo workspace data. |
| `npm run db:refresh` | Reset demo data, then seed again. |

## Core Rules

- A Supabase Auth user must have an active `profiles` row to use Expensea.
- Every business table is scoped by `team_id` and protected by RLS.
- Only approved and reimbursed expenses affect budgets, reports, analytics, public financial totals, and settlement balances.
- Money is stored and calculated as financial amounts, and display formatting must not silently round cents to whole numbers.
- Equal shared splits allocate cents deterministically so participant shares add back to the original expense amount.
